Parteien / Terminologie

ViaThinkSoft ist der Entwickler der Softwarelösung „OIDplus“.

OIDplus ist eine Softwarelösung zu organisieren, zu verwalten und Informationen zu veröffentlichen Object Identifiers ( im Folgenden als OID bezeichnet) und ihre Besitzer (nachfolgend als Registrierungsstelle oder RA ). Es kann auf jedem Server heruntergeladen und installiert werden. Eine bestimmte Installation von OIDplus wird als OIDplus-Instanz oder OIDplus-System bezeichnet . Betreiber von OIDplus-Instanzen sind normalerweise Softwareentwickler, Arbeitsgruppen, Unternehmen oder andere Organisationen, im Folgenden als Systembesitzer bezeichnet . Ein Systemadministrator ist eine Person, die die OIDplus-Software verwaltet und wartet, und normalerweise auch dieselbe Person, die die Hardware und Software des Servers verwaltet.

Objektkennungen sind in einer hierarchischen Struktur organisiert, in der RAs OIDs unter den von ihnen verwalteten OIDs an andere RAs delegieren. Im Kontext einer bestimmten RA und OID ist die „ Superior RA “ die RA, die die Root-OID einer bestimmten RA delegiert hat. Die Aussagen zu „Superior RAs“ in diesem Dokument gelten jedoch nur, wenn die Superior RA OIDplus zum Delegieren der OID verwendet.

Definition von serverseitig und clientseitig

OIDplus ist eine Webanwendung, die in den Programmiersprachen PHP (serverseitig) und HTML / JavaScript (clientseitig) geschrieben ist.

Serverseitig

Die PHP-Skripte von OIDplus werden über einen PHP-Skriptinterpreter verarbeitet, der auf einer Webserversoftware („HTTP-Webserver“) ausgeführt wird, z. B. „Apache“ oder „Nginx“. Die tatsächlichen Daten von OIDplus (OIDs, RAs,…) werden in einem DBMS (Database Management Software) gespeichert, z. B. „MySQL“, „MariaDB“ oder „PostgreSQL“. Das DBMS wird normalerweise auf demselben Server installiert, auf dem PHP / Apache installiert ist. Der Serveradministrator kann es jedoch auf jedem Computer installieren.

Client-Seite

Während der Benutzer OIDplus durchsucht, stellt die Serverseite dem Client-Computer HTML- und JavaScript-Inhalte bereit. Mit einer Webbrowsersoftware (wie „Chrome“ oder „Firefox“) werden diese Inhalte verarbeitet und auf dem Bildschirm angezeigt. Verschiedene JavaScript-Skripte sind entweder erforderlich, um OIDplus auszuführen, oder sie verbessern die Benutzererfahrung, z. B. indem OID-Beschreibungen neu geladen werden, ohne die gesamte Seite neu zu laden, während der Benutzer OIDplus durchsucht. Möglicherweise sind einige externe JavaScripts enthalten (Polyfill und ReCaptcha), die am Ende des Dokuments beschrieben werden.

OID-Informationen

Der Hauptzweck von OIDplus besteht darin, Informationen zu OID-Informationen anzuzeigen und zu verwalten.

Einige Attribute von Objektkennungen (wie z. B. ASN.1-Kennungen) können nur von der übergeordneten RA und nicht von der RA dieser OID geändert werden. Andererseits können einige andere Informationen (wie Titel und Beschreibung) nur von der RA geändert werden, die die OID besitzt, und nicht von der Superior RA. Der Systemadministrator kann alle Informationen bearbeiten.

Versteckte OID-Informationen

Eine übergeordnete RA kann das "versteckte" Flag auf eine von ihnen delegierte OID setzen. Dies bedeutet, dass die OID und alle nachfolgenden OIDs für die Öffentlichkeit unsichtbar sind. Die Öffentlichkeit kann jedoch die Existenz versteckter OIDs durch einen Hinweis „ XX OIDs sind versteckt. Loggen Sie sich ein, um sie zu sehen “.

Der Systemadministrator kann alle OIDs anzeigen, auch wenn sie ausgeblendet sind.

Ein Besitzer einer OID kann alle OIDs sehen, die auf seine Root-OID verweisen, auch wenn einige davon ausgeblendet sind.

Wenn Sie Eigentümer einer OID sind und Ihre OID vor der Öffentlichkeit verbergen möchten, wenden Sie sich an Ihre übergeordnete RA (bevorzugt) oder an den Systemadministrator und bitten Sie diese, Ihre OID als versteckt zu markieren. Bitte beachten Sie, dass dadurch auch alle OIDs ausgeblendet werden, die Sie an sich selbst oder andere RAs delegieren.

Einladung von RAs

Wenn eine OID von einer RA oder vom Systemadministrator erstellt wird, wird der Eigentümer der OID durch eine E-Mail-Adresse definiert (die öffentlich angezeigt wird). Daher ist es möglich, eine OID zu besitzen, auch wenn Sie nicht im OIDplus-System registriert sind.

Wenn die OID erstellt wird und die RA mit einer solchen E-Mail-Adresse nicht vorhanden ist, wird der Ersteller gefragt, ob er eine Einladung senden möchte. Wenn sie zustimmen, wird eine E-Mail an diese E-Mail-Adresse gesendet, sodass sie auf einen Link klicken können, um sich beim OIDplus-System zu registrieren, damit sie weitere Kontaktdaten hinzufügen sowie OID-Informationen delegieren und bearbeiten können.

Derzeit kann sich eine RA nicht registrieren. Sie benötigen eine Einladung des Systemadministrators oder ihrer übergeordneten RA, um sich beim OIDplus-System registrieren zu können. (Hinweis: Um eine Einladung erneut zu senden, muss der Systemadministrator oder die übergeordnete RA in der OID-Delegierungstabelle auf "Aktualisieren" klicken, damit sie erneut aufgefordert werden, die Einladung zu senden.)

Zusätzlich zum Einladungssystem kann der Systemadministrator RA-Konten ohne Überprüfung der E-Mail-Adresse erstellen.

Veröffentlichen von RAs / OIDs auf oid-info.com

oid-info.com ist eine Webseite, auf der Informationen zu OIDs wie in einem Wiki gesammelt und angezeigt werden. OIDplus enthält Funktionen zum Übermitteln von Daten seiner Datenbank an oid-info.com

Es gibt zwei Möglichkeiten:

  1. Manuelle Übertragung: Der Systemadministrator kann eine XML-Datei erstellen, die OID- und RA-Informationen enthält (nur die Teile, die ohnehin für die Öffentlichkeit sichtbar sind) und diese XML-Datei auf oid-info.com hochladen
  2. Wenn das OIDplus-System bei ViaThinkSoft registriert ist und die Übertragung von OID-info.com aktiviert ist (siehe unten), ruft ViaThinkSoft die XML-Daten automatisch über eine verschlüsselte Schnittstelle vom OIDplus-System ab und sendet die Daten an den Administrator von oid-info.com .

Eindeutige System-ID

Während der Ersteinrichtung wird ein öffentliches / privates Schlüsselpaar erstellt, das für verschiedene Zwecke verwendet werden kann, z. B. um Daten zwischen OIDplus und ViaThinkSoft zu verschlüsseln, wenn der Dienst "Systemregistrierung" verwendet wird (siehe unten) oder um OID- zu signieren. Over-WHOIS-Antworten.

Aus diesem öffentlichen Schlüssel wird eine eindeutige System-ID abgeleitet. Die eindeutige System-ID und der öffentliche Schlüssel werden im Setup-Bildschirm angezeigt.

Systemregistrierung und automatische Veröffentlichung auf oid-info.com

Während der Ersteinrichtung kann der Systemadministrator zwischen folgenden Optionen wählen:

0 Registrieren Sie sich beim Verzeichnisdienst und veröffentlichen Sie automatisch RA / OID-Daten unter oid-info.com.
1 Registrieren Sie sich nur beim Verzeichnisdienst.

Diese Einstellung kann jederzeit im Administrator-Login / Control Panel geändert werden.

Registrieren des Systems beim Verzeichnisdienst (Option 0 und 1)

Die Registrierung Ihres OIDplus-Systems hat verschiedene Vorteile:

  • Der öffentliche Schlüssel Ihres Systems wird veröffentlicht, damit Benutzer die Integrität Ihrer Daten überprüfen können (z. B. signierte OID-over-WHOIS-Anforderungen).
  • Da die 31-Bit-System-ID Hash-Konflikte aufweisen kann, stellt ViaThinkSoft sicher, dass die System-ID nur von Ihrem System verwendet wird (andere OIDplus-Systeme mit derselben System-ID werden gezwungen, ihre IDs zu ändern).
  • ViaThinkSoft kann den Systemnamen, die Administrator-E-Mail und die OID-Stammbögen bestätigen. Daher kann Ihr System mithilfe der Überweisung WHOIS gefunden werden.
  • Optional können Sie die automatische Veröffentlichung Ihrer öffentlichen OID-Informationen im Repository oid-info.com aktivieren.
  • Ihre OIDplus-Version wird an ViaThinkSoft gemeldet. Wenn ein schwerwiegender Fehler oder eine Sicherheitslücke Ihr System betrifft, kann ViaThinkSoft Sie per E-Mail warnen.
  • Der Verzeichnisdienst erstellt automatisch ein X.509-Sicherheitszertifikat für Sie (KEIN SSL-Zertifikat!), Das für verschiedene Überprüfungen verwendet werden kann.

Protokolldateien

Protokolle von OIDplus

OIDplus verwaltet Protokolleinträge für bestimmte Ereignisse wie Anmeldungen, OID / RA-Änderungen, Erstellen und Löschen von OIDs usw.

Diese Protokollnachrichten werden nur im DBMS gespeichert.

Alle Protokollereignisse sind für den Systemadministrator sichtbar.

Einige Ereignisse sind für die RA und / oder die Superior RA sichtbar.

Protokollereignisse zeigen möglicherweise die IP-Adressen des Computers an, der mit dem OIDplus-System verbunden war, und die E-Mail-Adresse, wenn die RA mit ihren Anmeldeinformationen angemeldet war.

OIDplus bietet keine Funktion zum Löschen von Protokollereignissen. Ein Löschvorgang muss direkt in der Datenbank erfolgen, z. B. mit einem Datenbankclient wie phpMyAdmin.

Protokolle von Host-Software, Firewalls usw.

Neben von OIDplus erstellten Protokollen kann auch andere clientseitige oder serverseitige Software, die an der Verarbeitung der Webanforderungen beteiligt ist (z. B. der HTTP-Webserver, der PHP-Skriptinterpreter, der Datenbankserver, das Betriebssystem, der Webbrowser oder verschiedene Firewalls) Protokollieren Sie Datenverkehr und Abfragen auf verschiedene Arten. Diese Protokolle sind unabhängig von OIDplus und müssen vom Systemadministrator und / oder den Netzwerkadministratoren verwaltet werden.

Wenn Sie über den Inhalt der Protokolldateien besorgt sind, wenden Sie sich an den Systemadministrator.

Externe JavaScripts

Während des Ladens der Webseite werden verschiedene JavaScripts hinzugefügt, um die Benutzererfahrung zu verbessern und wichtige Funktionen hinzuzufügen. Die grundlegenden Skripte sind die JavaScript-Dateien von OIDplus selbst, die auf dem Server der jeweiligen OIDplus-Installation gespeichert sind.

Es gibt zwei JavaScripts, die von einer externen Quelle / einem externen Server geladen werden:

Polyfill (nur wenn Internet Explorer verwendet wird)

Um OIDplus mit dem Webbrowser „Internet Explorer“ kompatibel zu machen, enthält OIDplus ein JavaScript-Skript von „polyfill.io“, einem Dienst der Financial Times.

Weitere Informationen finden Sie in den Datenschutzbestimmungen und Nutzungsbedingungen .

Das Polyfill-JavaScript ist nur enthalten, wenn OIDplus erkennt, dass der Webbrowser Internet Explorer ist.

Um die Polyfill-Funktion zu deaktivieren, muss der Systemadministrator die folgende Zeile in die Konfigurationsdatei "userdata / baseconfig / config.inc.php" im Ordner der OIDplus-Installation einfügen.

OIDplus::baseConfig()->setValue('DISABLE_MSIE_COMPAT', true);

Das Hinzufügen dieser Zeile auf der Serverseite und / oder das Blockieren des JavaScript auf der Clientseite führt dazu, dass die Seite im Webbrowser „Internet Explorer“ nicht funktioniert.

ReCAPTCHA (nur wenn "ReCAPTCHA" aktiviert ist)

Um vertrauliche Funktionen von OIDplus (z. B. das Anmeldeformular) vor Hackern zu schützen, kann der Systemadministrator "ReCAPTCHA" aktivieren, einen Dienst von Google. Vor dem Aktivieren von ReCAPTCHA in den OIDplus-Einstellungen muss der Systemadministrator einen API-Schlüssel von Google anfordern. Die Verwendung von ReCAPTCHA wird wahrscheinlich zusätzliche Cookies einführen, die außerhalb der Kontrolle von OIDplus liegen.

Weitere Informationen finden Sie in den Datenschutzbestimmungen und Nutzungsbedingungen von Google .

Wenn der Systemadministrator die ReCAPTCHA-Funktion nicht aktiviert hat, wird überhaupt kein externes Skript eingefügt.

Wenn die Funktion aktiviert ist, führt das Blockieren des ReCAPTCHA-JavaScript auf der Clientseite dazu, dass verschiedene Funktionen (wie das Anmeldeformular) nicht funktionieren.

Zusätzliche Sicherheits- / Zugriffskontrollen

Systemadministratoren können andere Methoden auswählen, um den öffentlichen Zugriff zu deaktivieren, z. B. mithilfe von Firewalls (sodass Clients überhaupt nicht auf den Server zugreifen können) oder indem Sie einen Kennwortschutz auf der Ebene der HTTP-Webserversoftware einrichten (z. B. "Basic Auth") Apache, damit die Clients nicht auf das Verzeichnis zugreifen können, in dem OIDplus installiert ist).

Verschlüsselte Netzwerkverbindungen

Der Serveradministrator ist für die Sicherung der Netzwerkverbindungen mit „HTTPS“ verantwortlich. Dies erfolgt auf HTTP-Webserverebene (dh einem Apache-Modul / einer Apache-Einstellung) und unabhängig von OIDplus.
